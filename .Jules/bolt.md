## 2026-04-27 - Uint8Array and Binary String Conversion Bottleneck
**Learning:** For large typed arrays (`Uint8Array`) generated via compression or raw stream parsing, using `Array.from(bytes, byte => String.fromCharCode(byte)).join('')` or `Uint8Array.from(str.split('').map(char => char.charCodeAt(0)))` introduces extreme memory allocation and CPU overhead due to implicit intermediate string/array generation. In modern V8, these take hundreds of milliseconds per megabyte.
**Action:** Replace `Array.from` iterations with chunked `String.fromCharCode.apply` execution logic with a limit around 32768 to prevent stack overflow. Replace `split().map()` patterns with direct `charCodeAt` loops operating on pre-allocated `Uint8Array` buffers, which execute an order of magnitude faster.
