## 2024-05-24 - Parallelizing File Processing
**Learning:** For resource-heavy client-side loops (e.g., processing multiple large PDFs in `iniciarLote`), replacing sequential `await` loops with a worker-pool pattern and a concurrency limit (e.g., 3) maximizes throughput while avoiding browser memory exhaustion or UI thread starvation. Refactoring worker functions to return individual results preserves deterministic output order when assigning results by their original index to a global array.
**Action:** When implementing parallel asynchronous processing for collective state, always refactor workers to return results and map them to a temporary array using their original indices before appending to globals, and preserve global array references by mutating in-place (`length = 0`) instead of reassigning.
