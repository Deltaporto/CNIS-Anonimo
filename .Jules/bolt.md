## 2024-05-17 - Typed Array String Conversions in V8

**Learning:** Replacing `.join('')` on `Array.from()` map chains and replacing `Uint8Array.from()` with `.split('').map(...)` mapping chains for string/typed array conversions reveals huge performance bottlenecks in V8 for large arrays. For example, converting a large `Uint8Array` to a string using `Array.from(bytes, byte => String.fromCharCode(byte)).join('')` takes hundreds of milliseconds, whereas chunked `String.fromCharCode.apply` completes in a fraction of the time. Similarly, replacing chained iteration for converting strings back to bytes with a dedicated loop using bounded bitwise logic (`& 0xff`) creates significant efficiency gains.
**Action:** When performing string/byte conversions in V8, specifically to map characters to integers and vice versa on very large arrays, always implement chunked loops (like `bytesToString` using `String.fromCharCode.apply(null, chunk)` limiting to ~32K chunks to avoid call stack exceptions) and direct index array mapping via single bounded `for` loops (like the `encodeLatin1` function in `js/pdf-processor.js`).
