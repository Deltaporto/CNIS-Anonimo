## 2024-05-18 - Replacing Array.from with Chunked String.fromCharCode for Large Buffers
**Learning:** For extremely large `Uint8Array` buffers (like raw PDF bytes or full object streams), mapping over every byte with `Array.from(bytes, byte => String.fromCharCode(byte)).join('')` is a major performance bottleneck due to excessive array allocations and function calls. A custom chunked approach using `String.fromCharCode.apply` processes data in manageable blocks without blowing the call stack and avoids the massive overhead of intermediate arrays.
**Action:** When converting large buffers or arrays to binary strings, replace `Array.from()` mappings with a helper function that chunks the array and processes it using `String.fromCharCode.apply` with a max chunk size of `32768`.
