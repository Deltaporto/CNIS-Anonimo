## 2026-04-24 - Dynamic ARIA progress bars
**Learning:** When building custom DOM-based components like progress bars (e.g., using `div` elements for tracking file processing), static initial setup is not enough. The `aria-valuenow` attribute must be updated synchronously in Javascript alongside the visual progress indicator (like a CSS width percentage) so that screen readers correctly announce real-time loading state updates to users.
**Action:** Always map dynamic visual attributes (like `style.width`) in UI updating functions to their semantic ARIA equivalents (like `aria-valuenow`) during custom component updates.
