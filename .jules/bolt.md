## 2024-05-04 - Fast Latin1 encoding and decoding arrays
**Learning:** In modern V8 environments, the chunked `String.fromCharCode.apply` helper (`decodeLatin1`) is measured to be significantly faster (~10x-100x) than naive array splitting and mapping for converting large binary `Uint8Array`s to strings. It handles byte-to-char mapping correctly over Windows-1252 character mapping issues (e.g. byte 128) inherent to most browser/Node `TextDecoder` implementations of Latin1. Also, a fixed buffer loop (`encodeLatin1`) is drastically faster than `Uint8Array.from(str.split('').map(char => char.charCodeAt(0)))`.
**Action:** Use chunked `String.fromCharCode.apply` (`decodeLatin1`) and fixed buffer loops (`encodeLatin1`) instead of `Array.from` with `.split('').map()` iteration and joining when dealing with raw binary strings and Uint8Arrays.
