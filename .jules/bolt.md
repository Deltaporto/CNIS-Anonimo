## 2025-02-28 - Avoid Array.from with functional maps over typed arrays
**Learning:** In V8/Node.js, executing `Array.from(uint8Array, byte => String.fromCharCode(byte)).join('')` on large typed arrays (like decompressed PDF streams) causes massive performance degradation and potential `Maximum call stack size exceeded` errors. The mapping function overhead when processing millions of bytes is enormous. Alternatively, calling `.split('').map(char => char.charCodeAt(0))` for string to array conversion suffers from the same overhead.
**Action:** Replace `Array.from()` maps on `Uint8Array` with a chunked array processor (`String.fromCharCode.apply(null, chunk)` with 32768 sized chunks), and replace string mapping patterns with direct `Uint8Array` allocation and loop assignment. This approach reduces execution time from seconds to milliseconds.
